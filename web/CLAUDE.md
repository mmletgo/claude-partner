# web/ - React 前端

## 概述

基于 React + TypeScript + Vite 的前端界面，宿主为 **Tauri 2**，通过 `invoke()` IPC 调用 Rust 后端命令（`src-tauri/`）。迁移自 PyQt + aiohttp，前端已无任何本地 HTTP 调用。

## 开发命令

- `npm run dev` — 启动 Vite 开发服务器（端口 5173）
- `npm run build` — 打包到 dist/（tsc 类型检查 + vite 构建）
- `npm run lint` — ESLint 检查
- `npx playwright test tests/screenshot-overlay.spec.ts --project=chromium` — 截图 Overlay 工具条时序回归测试（会自动拉起 Vite）
- `npx --yes tsx src/pages/Settings/HealthPanel.test.ts && npx --yes tsx src/pages/Settings/shortcutRecorder.test.ts && npx --yes tsx src/pages/Settings/settingsState.test.ts` — Settings 健康提醒布局/时间选择、快捷键录制与状态/payload 回归测试
- `npx --yes tsx src/lib/workbenchDependency.test.ts && npx --yes tsx src/components/domain/WorkbenchProjectRail/projectSessionStats.test.ts && npx --yes tsx src/hooks/workbenchTerminalBuffer.test.ts && npx --yes tsx src/api/promptOptimizer.test.ts && npx --yes tsx src/pages/Workbench/workbenchWorktrees.test.ts && npx --yes tsx src/pages/Workbench/workbenchFiles.test.ts && npx --yes tsx src/pages/Workbench/workbenchRemoteProjects.test.ts && npx --yes tsx src/components/domain/WorkbenchHtmlPreview/htmlAssets.test.ts && npx --yes tsx src/components/domain/WorkbenchCodeEditor/workbenchCodeEditorTheme.test.ts && npx --yes tsx src/pages/Workbench/workbenchLayerStyles.test.ts && npx --yes tsx src/pages/Workbench/workbenchWorkspaceSwitch.test.ts && npx --yes tsx src/pages/Workbench/terminalSizing.test.ts && npx --yes tsx src/pages/Workbench/terminalSessionOrder.test.ts && npx --yes tsx src/pages/Workbench/terminalOptions.test.ts && npx --yes tsx src/pages/Workbench/terminalReplay.test.ts && npx --yes tsx src/pages/Workbench/promptOptimizerWidget.test.ts` — Workbench tmux 依赖状态 helper、项目卡片 window/pane 统计、跨路由终端输出缓存、Prompt 优化 API 单语/流式写入参数、worktree 过滤/状态映射、文件类型/能力矩阵/tab reducer、远端项目去重置顶/路径上级/目录排序 helper、HTML 预览相对资源重写、代码编辑器主题 token/One Dark Pro 语法与 YAML 高亮主题、文件/终端叠层样式、文件/终端双向切换契约、终端 pane/viewport 尺寸计算、多 terminal window 挂载顺序、xterm 主题 token 与 PTY/tmux 输出配置、历史 replay 输入屏蔽与 Prompt 优化项目上下文/填入规则回归测试
- 完整开发（前端 + Rust）：仓库根 `./web/node_modules/.bin/tauri dev`（自动拉起 vite + cargo run + 热重载）

- **API 模块**: window/pane、当前工作区文件树和 Git 提交树命令；remote API 负责远端 roots/listDir/info/openProject 项目选择入口，远端项目打开后继续复用同一套 worktrees/git/sessions/files API，后端按 remote project/worktree/session id 代理到远端设备；worktrees API 负责 `list/create/commit/push/merge/remove`，git API 负责 `listCommits(projectId, worktreeId?, limit?)` 并按 active worktree 返回提交 DAG、parent hashes 和 local/remote/tag refs，sessions API 名保留历史命名，语义上 `create/list/focus/focused/rename/close` 操作 terminal window，`create(projectId, initialSize, worktreeId)` 会把 window 绑定到 active worktree cwd，`focus(sessionId)` 会同步底层 tmux `optimizedZh/optimizedEn`，Workbench 快捷键小组件传 active worktree 根目录让后端 Claude Code 加载该工作区 CLAUDE.md，同时传设置页选择的 `zh|en` 和 active sessionId，由后端把单语优化结果流式写入终端；远端 sessionId `get_scratchpad_page` / `create_scratchpad_page` / `update_scratchpad_page_content` / `rename_scratchpad_page` / `delete_scratchpad_page` / `sync_scratchpad`）。`githubTrending.ts` 封装 Github热门页 GitHub Trending 5 命令：`list` / `getConfig` / `getDefaultConfig` / `updateConfig` / `testClaudeCli`；`config.ts` 的 `configApi` 封装当前配置 `get`、偏好默认值 `getDefaults`、配置更新 `update`（含基础偏好、截图快捷键、Workbench Prompt - **Workbench HTML/Markdown 预览资源**: `workbench.ts` files API 包含 `previewHtmlAsset(projectId, documentPath, assetPath, worktreeId?)`，对应后端 `preview_workbench_html_asset`；前端只把当前 worktree 根内相对资源交给后端，外链、data/blob、绝对路径和加载失败资源在 HTML srcDoc 或 Markdown 图片 DOM 中置空。Markdown 预览通过 Tiptap Image extension 渲染图片节点，DOM 初始 `src` 为空，资源加载成功后才替换为 data URL，避免把 data URL 写回源 Markdown。

- **Workbench HTML 预览**: HTML tab 的 source/preview/split 保持 sandbox iframe；进入 preview/split 前会重写核心 `src`/`href`/`poster`/`srcset` 资源为 data URL，并支持外部/内联 CSS 的相对 `url()` 二次重写。资源路径以当前 HTML 或 CSS 文件为基准解析，后端只读且限制在 active worktree 根内。Markdown tab 的 WYSIWYG/split 图片同样使用该资源通道，但只改 DOM 展示，不改源 Markdown 字符串。
- **Workbench 导航约定**: 终端和文件预览顶部栏复用 `WorkbenchWorkspaceNav`；终端文件预览按钮位于工具按钮组最右侧，文件预览 tab 标签显示相对路径，文件操作按钮与 tab 同行显示。
- **Workbench 代码编辑器主题**: `WorkbenchCodeEditor` 必须通过 `theme` prop 显式传入 token 化 CodeMirror theme，编辑器背景、左侧行号 gutter、当前行和搜索/面板 chrome 跟随应用主题 token；语法高亮保留 One Dark Pro 色系，避免 `@uiw/react-codemirror` 默认 light theme 把暗色工作台刷成白底。
- **Workbench Merge UI**: Git 历史 Merge 按钮只要求 active worktree 非主且无其他 worktree 操作；不要用可能过期的 `status.clean` 禁用。点击后由后端 `merge_workbench_worktree` 实时检查源 worktree dirty 状态，并通过 `workbench:merge-progress` 推送带 `projectId` 的 `checkSource/closeSessions/mergeMain/resolveConflicts/cleanup` 阶段；前端只接收当前项目事件，在 Git 历史工具条下展示阶段条，命令结束后刷新 worktrees、sessions、Git history，并移除已关闭 worktree 的 terminal buffer。成功完成的阶段条只短暂展示后自动隐藏，失败阶段保留给用户查看错误。
- **Workbench 终端性能约定**: 终端输出缓存必须用 session 级外部 store，`TerminalPane` 通过 `useWorkbenchTerminalBuffer(sessionId)` / `useSyncExternalStore` 只订阅自身 session；禁止把 `workbench:terminal-output` 转成 Context state 后驱动整个 Workbench 页面重渲染。终端面板 CSS 与 xterm theme 都必须读取 `--terminal-*` token，浅色/深色主题 token 必须分开，已挂载 xterm 在 `cp-theme-change` / `storage` 后同步 `terminal.options.theme`。Prompt 优化浮层关闭时只缓存终端光标锚点到 ref，不提交位置 state；只有浮层打开且位置变化时才更新浮层坐标，避免终端输出/滚动期间高频 cursor move 卡住页面。
- **Workbench worktree 创建**: `/workbench` 顶部“新建 worktree”先展开页面内表单，分支前缀从固定类型（`feature`/`fix`/`chore`/`docs`/`refactor`/`test`/`hotfix`）中选择，用户只输入后缀，确认后组合为 `<prefix>/<suffix>` 并调用 `create_workbench_worktree`；不要使用 `window.prompt` 等 WebView 阻塞弹窗，否则桌面环境可能没有可见反馈。
- **路由**: `src/App.tsx` — React Router，`/` → Home，`/prompts` → Prompts，`/cc-history` → CcHistory，`/workbench` → Workbench，`/scratchpad` → Scratchpad，`/prompt-optimizer` → PromptOptimizer，`/claude-md` → ClaudeMd，`/ssh` → SSH，`/health` → Health 等

## i18n 国际化

- **库**: react-i18next + i18next，初始化在 `src/i18n/index.ts`（`declare module` 类型扩展，`t()` 的 key 编译期校验，拼错即 tsc 报错）
- **命名空间**: `common`（动作/状态枚举/方向，跨页共享）、`nav`、各页面一个（home/prompts/ccHistory/transfer/devices/scratchpad/promptOptimizer/claudeMd/welcome/settings/ssh/health/workbench）
- **写法约定**（i18next v26 类型硬要求）:
  - 组件内 `const { t } = useTranslation([用到的所有ns数组]);`，所有 `t('ns:key')` **带 ns 前缀**
  - 模块级 helper 接收 `t: TFunction<'ns'>`，内部调用**省略 ns 前缀**
  - 插值 `{{var}}`；英文复数 en `{key}` + `{key}_other`，zh 只 `{key}`
- **切换/持久化**: `useLanguage` hook（localStorage `cp-lang` + `cp-lang-change` 自定义事件 + `storage` 事件跨标签/多窗口同步）；首次按系统语言推断
- **术语保留英文**: Prompt / GitHub / KB·MB 等在 en/zh 资源都保留英文
- **DesignSystem 页**: dev-only，不纳入 i18n（保持英文）
- **新增页面文案**: 在 `src/i18n/locales/{en,zh}/<页ns>.json` 加 key + 组件 `useTranslation(['<页ns>','common'])` + `t('<页ns>:key')`；改完 `npm run build`（tsc 校验 key）
