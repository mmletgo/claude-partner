# 前端 i18n 多语言改造设计

> 状态:已确认,待实现
> 日期:2026-06-11
> 范围:`web/` 前端(React 19 + TypeScript + Vite)

## 1. 目标

为前端界面引入 i18n 多语言切换,支持 **中文 / 英文**,默认 **跟随系统语言**(中文系统→中文,其他→英文),用户手动切换后记住选择。改造范围限定 `web/` 前端,**后端与 PyQt 宿主零改动**。

## 2. 已确认决策

| 决策项 | 选择 | 理由 |
|---|---|---|
| i18n 库 | react-i18next | React 生态事实标准,复数/插值/命名空间/懒加载成熟,类型扩展支持 |
| 语言持久化 | localStorage(纯前端) | 复刻现有 `useTheme` 范式,不改后端 |
| 切换入口 | Sidebar 底部全局 LanguageSwitcher | 任意页面随手切换 |
| 首次语言 | 跟随系统(`navigator.language`) | zh 开头→中文,否则→英文;用户切换后写入 localStorage 记住 |
| 默认回退 | 英文(en) | `fallbackLng='en'` |
| 资源组织 | 多 namespace | 文件小、可按需懒加载、便于并行 subagent 改造 |
| DesignSystem 页 | 跳过翻译 | dev-only,保持英文硬编码 |
| 技术术语 | 保留英文 | Prompt / GitHub / KB·MB 等在中英文资源里都保留英文 |

## 3. 目录结构

```
web/src/i18n/
├── index.ts          # i18next 实例初始化(资源导入/语言检测/fallback/插值/类型扩展)
├── useLanguage.ts    # 语言 hook(localStorage 持久化 + 自定义事件跨标签同步)
└── locales/
    ├── en/{common,nav,home,prompts,transfer,devices,scratchpad,welcome,settings}.json
    └── zh/{...同构,不创建 designSystem}
```

## 4. i18n 初始化 (`i18n/index.ts`)

- 同步 `import` 导入全部 ns 的 en/zh 资源(项目规模不大,无需懒加载,简化实现)
- `fallbackLng='en'`
- **不引入** `i18next-browser-languagedetector`,自己写检测逻辑以复刻 `useTheme` 轻量范式:
  - 优先级:`localStorage['cp-lang']` > 系统语言(`navigator.language` 以 `zh` 开头→`'zh'`,否则→`'en'`) > `'en'`
- 插值配置:`{{var}}` 语法(i18next 默认)
- 复数:依赖 i18next 内置 `_one`/`_other` 后缀规则(中文不分单复数,zh 资源只需主 key)
- **类型扩展**:`declare module 'i18next'`,让 `react-i18next` 的 `t()` 在编译期校验 key,拼错即 `tsc` 报错
- 在 `web/src/main.tsx` 顶部、`createRoot().render()` 之前 `import './i18n'`(副作用初始化)

## 5. 语言切换机制

### 5.1 `useLanguage` hook(复刻 `useTheme`)

- 读取 `localStorage['cp-lang']`;无值时按系统语言推断并写入
- 暴露 `{ language: 'en' | 'zh'; setLanguage(lang) }`
- `setLanguage(lang)`:`i18next.changeLanguage(lang)` + 写 localStorage + `window.dispatchEvent(new CustomEvent('cp-lang-change'))`
- 监听 `'cp-lang-change'` 与 `'storage'` 事件,实现跨标签页 / 多 QWebEngineView 窗口同步
- 语言变化后由 `react-i18next` 的 `useTranslation` 触发组件重渲染

### 5.2 `LanguageSwitcher` 组件

- 位置:`web/src/components/layout/LanguageSwitcher.tsx`,嵌入 `Sidebar` 底部(与现有 `ThemeToggle` 并列或邻近)
- 形态:紧凑切换器,显示当前语言并提供 中 / EN 切换(图标 + 当前语言标识)
- 点击即调 `setLanguage`,无确认弹窗

## 6. 翻译改造规范

| 形态 | 改造方式 |
|---|---|
| 静态文本 | `<h1>{t('settings:title')}</h1>` |
| 动态插值 | `t('transfer:activeCount', { n })` → en `"{{n}} active"`、zh `"活跃 {{n}}"` |
| 英文复数 | `t('home:devicesOnline', { count })`<br>en:`"devicesOnline": "{{count}} device online"` + `"devicesOnline_other": "{{count}} devices online"`<br>zh:`"devicesOnline": "{{count}} 台在线"` |
| 枚举状态表 | `StatusDot` / `TransferItem` 的 `status→label` map 改为 `t('common:status.transfer.pending')` 等,集中到 `common` |
| aria-label / title | 一并翻译(`PromptCard` / `Tag` 等) |

## 7. `lib/format.ts` 改造

- `toLocaleDateString('zh-CN')` 硬编码 → 按当前语言动态选 locale(`'zh'`→`'zh-CN'`,否则 `'en-US'`)
- `formatRelativeTime` 改用 `Intl.RelativeTimeFormat(lang, { numeric: 'auto' })`,删除手写「刚刚 / 分钟前 / 小时前 / 天前」
- 统一 `DeviceCard` 内重复的 `formatLastSeen` 到 `format.ts`(复用而非重写)
- format 函数签名增加 `lang?: 'en' | 'zh'` 参数,缺省取当前 i18next 语言

## 8. namespace 划分

| ns | 内容 |
|---|---|
| common | 动作(复制/删除/保存/取消/确认/刷新/新建)、状态(在线/离线/忙碌/离开)、传输状态枚举、通用空态/搜索 placeholder |
| nav | 侧栏导航项 |
| home | 首页 |
| prompts | Prompt 库页 |
| transfer | 文件传输页 |
| devices | 设备页 |
| scratchpad | 速记本页 |
| welcome | 欢迎引导页 |
| settings | 设置页(文案最多) |

## 9. 改造范围

生产可见:

- 页面:`Settings`、`Transfer`、`Prompts`、`Home`、`Welcome`、`Devices`、`Scratchpad`
- 组件:`TransferItem`、`DeviceCard`、`PermissionCard`、`TagInput`、`PromptCard`、`Tag`、`StatusDot`
- 工具:`lib/format.ts`

跳过:`DesignSystem` 页(dev-only,保持英文硬编码)。

## 10. 错误处理 / 降级

- 缺失 key:i18next 回退到 key 字符串本身 + 开发期 console 警告
- 缺失语言资源:回退 `en`
- 类型扩展保证 key 拼写错误在编译期(`tsc -b`)报错

## 11. 后端 / 宿主影响

无。语言是纯前端展示层,后端 `config` / API 无 locale 字段,QWebEngineView 嵌入的 React 自动跟随语言切换。
